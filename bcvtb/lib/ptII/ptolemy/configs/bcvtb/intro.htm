The Building Controls Virtual Test Bed (BCVTB) is a software environment that allows expert users to couple different simulation programs for distributed simulation. For example, the BCVTB allows to simulate a building and HVAC system in EnergyPlus and the control logic in MATLAB/Simulink, while exchanging data between the software as they simulate. The BCVTB is based on the Ptolemy II software environment. The BCVTB is still under development and aimed at expert users of simulation. Due to the different programs that may be involved in distributed simulation, familiarity with compiling and configuring programs is essential.

Demonstrations

Note: Some demonstrations require to be started from dos or bash. Otherwise, files with relative path names may not be found.

Windows

  • dymola-room: Dymola simple room model, controls simulated by Ptolemy.
  • dymolaSimulink-room: Dymola simple room model, controls simulated by Simulink.
  • ePlus31-simple: EnergyPlus building and HVAC system, setpoint simulated by Ptolemy.
  • ePlus31Ptolemy-vav: EnergyPlus building, variable air volume flow system simulated by Ptolemy.
  • ePlus31Simulink-simple: EnergyPlus building and HVAC system, setpoint simulated by Simulink.
  • matlab-room: Matlab simple room model, controls simulated by Ptolemy.
  • simulink-room: Simulink controller, simple room model simulated by Ptolemy.
  • ptolemy-room: Room model, simulated by Ptolemy.
  • ptolemy-room: Room model with finite state machine, simulated by Ptolemy.
  • ptolemy-room: Room model with icon color changing based on room temperature, simulated by Ptolemy.
  • c-room: Room model simulated in C, controls simulated by Ptolemy.
  • c-timeSync: Two programs implemented in C that exchange their data.
  • f90-room: Room model simulated in F90, controls simulated by Ptolemy.
  • www-weather: Example that gets minimum and maximum outside temperature from Berkeley weather forecast.
